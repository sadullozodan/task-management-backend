## Decisions Made

- **Re-invite invalidates prior pending token** (delete-on-re-invite) rather than letting multiple live tokens exist for the same email+workspace. Simpler for the frontend — always use the freshest link.
- **Existing-user invite returns 409** (not silently adding them). The add-existing-user path (`POST /members`) remains separate; the invite path is strictly for not-yet-members.
- **Email match enforced at accept time** (user.email must match invite.email, case-insensitive) to prevent token forwarding. Same check applied at register-with-invite time.
- **Production email stub throws** rather than silently dropping the email. Forces implementors to wire up a real provider before going live.
- **`app.prisma`** used in tests for direct DB manipulation (expired-invite fixtures), consistent with the rest of the test suite.
