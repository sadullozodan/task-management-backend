## Decisions Made

- **Refresh token as JWT (not opaque):** Used JWT with `jti` = DB row ID as the refresh token. The `token_hash` column (designed for opaque tokens) stores a random UUID as a unique placeholder. This avoids hashing and simplifies lookup — verification is `verifyRefreshToken(rawToken)` → `prisma.refreshToken.findUnique({ where: { id: payload.jti } })`.
- **`loadDotEnv` skips in test mode:** Without this, Vitest setup's `DATABASE_URL ??=` would be overridden when config is first imported (because `process.loadEnvFile()` overwrites existing vars). Fix: skip .env loading when `NODE_ENV=test`.
- **Integration tests use the dev DB (`taskmgmt`):** No separate test DB. Each test run uses a unique timestamp-prefixed email/slug and deletes created rows in `afterAll`. CI uses a fresh DB from the Postgres service, so no cleanup race conditions in CI.
- **`requireWorkspaceMember('owner')` for `DELETE /workspaces/:slug`:** Only the workspace owner can delete the workspace. Admin can update it.
- **`completed_at` set automatically:** When updating an issue's `state_id`, the service fetches the new state's group; if `completed`, it sets `completed_at = now()`; otherwise `null`. No separate endpoint.
